# portfolio-health-check

用于诊断用户已有持仓，而不是从零构建组合。先收集持仓、现金、基准、风险偏好和期限，再给出风险和改进优先级。

## 诊断框架

1. 持仓归一：把股票、现金和其他资产统一为组合权重。
2. 集中度：单票、前五大、行业、市场、市值、风格和资产类别集中度。
3. 相关性：识别 3 只以上走势高度相关的持仓群组，估算有效分散程度。
4. 因子暴露：Beta、价值、成长、规模、动量、质量、波动、股息率。
5. 风险指标：波动率、跟踪误差、主动份额、VaR/CVaR、最大回撤、夏普/索提诺。
6. 压力测试：2015 股灾、2018 贸易摩擦、2020 疫情冲击、2022 调整、2008 金融危机。
7. 流动性：日均成交额、退出天数、涨跌停风险、小盘股占比。

## 输出结构

- 健康评分 0-100，并说明数据完整度。
- 严重问题、警示事项、组合优势。
- 改进建议：按影响和可执行性排序。
- 调仓方案：只给研究建议和约束，不直接执行交易。
- 需要补充的数据清单。

## SATS 边界

如果用户没有提供完整持仓权重，应先说明假设或要求补充。不要基于空缺数据生成精确风险数值。
