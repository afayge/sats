# tech-hype-vs-fundamentals

用于对比科技公司叙事热度和真实基本面。重点不是判断概念真假，而是拆解市场定价中隐含的增长、利润率和政策预期是否合理。

## 分析框架

1. 公司阶段：亏损扩张、收入高增长、盈利爬坡、成熟平台，不同阶段使用不同估值指标。
2. 估值指标：PS、PE、PEG、EV/EBITDA、市值/收入、研发投入与收入比。
3. 增长质量：收入增速、扣非利润、毛利率、客户集中度、回款、现金流、研发资本化。
4. A 股科技特殊因素：政策溢价、国产替代、主题资金、解禁压力、科创/创业板波动。
5. 高估特征：估值远高于可验证增长、利润质量弱、概念多业务少、政策预期过满。
6. 低估特征：增长被短期扰动掩盖、现金流稳健、估值回落但竞争力未坏。

## 输出结构

- 关键发现摘要：高估、合理、低估或需更多数据。
- 估值 vs 基本面表。
- 市场正在定价什么，以及哪些假设最脆弱。
- 催化剂和风险事件：业绩、政策、订单、解禁、竞争格局。
- 合理估值区间只作为情景分析，不作为目标价承诺。

## SATS 边界

没有真实财务和估值数据时，只能输出框架和需要的数据项，不得用概念热度替代基本面证据。
