# Discover 工作流程分析

## 入口

`discover` 有 4 个入口表面：

| 入口 | 文件 | 关键行 |
|------|------|--------|
| CLI `sats discover` | `sats/cli.py` | L373-384 (parser), L1597-1652 (handler) |
| REPL `/discover` | `sats/repl.py` | L66, L122, L163 |
| Chat 自然语言 | `sats/chat_components.py` | L55, L1217, L1373-1387 |

## 两条路径

CLI handler (`cli.py:1597`) 根据是否提供自然语言 `query` 分支：

### 路径 A：纯信号发现（无 query）
直接调用 `run_opportunity_discovery()`

### 路径 B：自然语言选股（有 query）
调用 `run_stock_picking_agent()` → 内部包装 `run_opportunity_discovery()`

## 核心流水线 (`run_opportunity_discovery`)

**文件**: `sats/analysis/opportunity_discovery.py:355-601`

```
1. 解析交易日                    (L380)
2. 加载扫描输入 — 全 A 股或过滤后的股票池  (L386-420)
3. 信号分析 — 对每只股票运行 short_up 信号   (L422-432)
4. 热门板块上下文 — 识别热点板块/主题       (L433-450)
5. 本地候选筛选 — 按分数阈值过滤          (L452-457)
6. 多样化候选 — 避免信号结果过于集中       (L459-464)
7. 加载市场背景 — A 股大盘/指数数据       (L479-500)
8. 候选丰富 — 添加指标、关键位、热点分数、缠论分数 (L503-527)
9. LLM 排名 — 发送给 LLM 做最终排名       (L530-567)
10. 生成报告 — 写 Markdown 到 reports/    (L569-588)
11. 返回结果 — OpportunityDiscoveryResult  (L589-601)
```

## 选股 Agent 流水线 (`run_stock_picking_agent`)

**文件**: `sats/analysis/stock_picking_agent.py:337-502`

当提供自然语言 query 时：

```
1. 构建计划 — 从 query 提取 profile/theme/horizon/constraints  (L309)
2. 解析主题股票池 — 主题关键词映射到股票池 (同花顺/申万/LLM)  (L529)
3. 运行机会发现 — 带可选股票过滤的 run_opportunity_discovery() (L426-447)
4. 本地排名 — 基于 profile 的评分            (L450)
5. 因子叠加 — 可选多因子评分                 (L451-461)
6. Agent LLM 排名 — 带研究上下文的 LLM 排名   (L464-491)
7. 报告生成                                  (L492-493)
```

## 核心数据结构

- **`OpportunityCandidate`** (`opportunity_discovery.py:96`) — 单个候选股票，含 ts_code, name, score, ranking, trend, chan_signals, hot_sectors 等
- **`OpportunityDiscoveryResult`** (`opportunity_discovery.py:168`) — 完整结果，含 candidates, market_context, hot_sector_context, report_path 等
- **`StockPickingAgentResult`** (`stock_picking_agent.py:226`) — 包装上层结果 + query, plan, theme_universe, evidence

## 热点板块机制深度分析

### 常量配置 (`opportunity_discovery.py`)

| 常量 | 值 | 说明 |
|------|-----|------|
| `DEFAULT_HOT_SECTOR_DAYS` | 5 | 默认回看天数 |
| `HOT_SECTOR_SCORE_CAP` | 12.0 | 单股热点加分上限 |
| `DIVERSITY_PENALTY` | 2.0 | 多样化惩罚基数 |

### 数据结构

**`OpportunityCandidate`** 热点字段 (`opportunity_discovery.py:107-108`):
- `hot_sectors: list[dict]` — 该股所属热点板块列表
- `hot_sector_score: float` — 热点加分
- `ranking_score = local_score + hot_sector_score + chan_score`

**`hot_sector_context`** (`opportunity_discovery.py:175`):
```python
{
    "trade_date": str,
    "lookback_days": int,
    "hot_industries": list[dict],      # top-10 行业板块
    "hot_concepts": list[dict],        # top-20 概念板块
    "stock_hot_sectors": dict[str, list[dict]],  # ts_code -> 热点板块列表
}
```

### 板块热度评分算法

**函数**: `_score_hot_sectors()` (`tushare_provider.py:2078-2124`)

```
heat = ret_5d * 0.35 + ret_3d * 0.35 + latest_pct * 0.15 + up_days_5 * 1.0 + up_days_3 * 1.5
```

防脉冲惩罚: 若 `latest_pct > 5` 且 `up_days_3 <= 1`，扣减 `latest_pct * 0.4`。

### 热点加分公式

**函数**: `_hot_sector_score()` (`opportunity_discovery.py:1856-1863`)

```python
score = min(12.0, best_heat * 0.8 + (2nd_heat + 3rd_heat) * 0.25)
```

只取最热 3 个板块，上限 12.0。

### 流水线中的 4 个集成点

1. **候选初筛** (`_select_local_candidates` L923): `_signal_ranking_score()` 含热点加分
2. **多样化选择** (`_pick_diversified_signal_results` L949): 同热点板块惩罚 1.0
3. **候选丰富** (`_enrich_candidates` L978): 设置 hot_sectors, hot_sector_score, ranking_score
4. **LLM 提示**: `to_llm_context()` + `market_llm_context.py` 默认含 `hot_sectors` 维度

```
ths_index API → _load_ths_sector_basic() → DuckDB
ths_daily API → _load_ths_sector_daily() → DuckDB

```
